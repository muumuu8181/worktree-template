# GASのセキュリティ認識と知名度問題

## 🎯 鋭い指摘！まさにその通り！

### 企業での「Gmail問題」と同じ構図

```
企業メール：
├── 社員@gmail.com → ❌ アマチュア感
├── 社員@company.com → ✅ プロフェッショナル
└── でも機能は同じGmail

API：
├── script.google.com/macros/s/xxx → ❌ 素人感
├── api.company.com → ✅ プロフェッショナル
└── でも中身は同じかも
```

## 🔒 セキュリティの実態

### GASのセキュリティ問題（企業視点）

```
企業が嫌がる理由：
1. URLが「google.com」
   └─ 自社ドメインじゃない = 管理外

2. アクセスログ
   └─ 自社で完全管理できない

3. データの所在
   └─ Googleのサーバー = コントロール不可

4. 監査
   └─ SOC2とかの監査通りにくい

5. 退職者問題
   └─ 個人Googleアカウントだと引き継ぎ面倒
```

### AWS Lambda / GCF との違い

| 項目 | GAS | Lambda/GCF |
|------|-----|-----------|
| ドメイン | google.com固定 | 自社ドメイン可 |
| IAM | Google個人アカウント | 企業管理のIAM |
| ログ | 限定的 | CloudWatch完備 |
| VPC | なし | プライベート可 |
| 監査証跡 | 弱い | 完璧 |
| コスト | 無料 | 従量課金 |
